Queries

Most panels that list metarecords are driven by a query. There are two languages: the simplified query you usually type, and the normal DSL it expands into.

Simplified queries

The query box (for example in the metarecord list panel) accepts the simplified language. A few examples: 

genre=jazz
rating>3
fav genre=jazz
size>=100MB
modified younger 3d

Words separated by spaces mean AND; use OR (or +) for alternatives, ! for negation, and parentheses to group. Comparison operators mirror the normal DSL (= != < <= > >=); ~ is a regular-expression match. An empty query matches everything.

The normal DSL

Expansion is a pure, client-side transformation — the simplified text is rewritten into the normal DSL before it is sent. Some panels can show and even freeze the expanded DSL so you can edit it by hand. The normal DSL adds three-valued predicates (present / absent / unknown) and traversal (follows a reference, ->* over a TreeRef path).

The current simplified grammar

The simplified language is configurable: this is the grammar in effect (loaded at startup from ~/.config/metafolder/core/query-grammar). Edit that file and re-run metafolder-sync-config to change it.
