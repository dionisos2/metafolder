Metarecord list panel

Lists the metarecords matching a query, as a table or a grid. It is the main way to browse and bulk-edit a repository.

Finder (quick filter)

The search box at the top is the finder: a fast, incremental filter that narrows the list as you type. Press / to focus it. Each whitespace-separated word must appear as a substring, in order (“ordered substring”): scien fic matches science-fiction, and conf gui matches a path …/config/…/gui/…. It searches several fields at once (shown at the right of the box): a TreeRef field like mfr_path is matched over its whole path, other fields over their text. The finder is combined with (AND-ed onto) the query box below, so you keep a structural query (e.g. type=tag) while typing to find within it. Change the searched fields with the metarecord-list:finder-fields workspace variable — a list of field:path / field:direct entries (a bare field auto-detects its mode); the default is mfr_path:path label:direct name:direct.

Query box

Type a simplified query and press Enter (or Apply) to run it; an empty query lists everything. Show normal DSL reveals the expanded query, which you can freeze to edit by hand. Focus it with Ctrl+/.

Columns

The columns box chooses what each row shows: space- or comma-separated field names. Suffixes change how a value is displayed:

  • field — the raw value.
  • field:mode — project a TreeRef value: :name (leaf), :uuid (parent), :path (full path from the root), :raw (the default parent/name couple).
  • field>sub — follow a Ref to the target's sub field (e.g. tag>label, or tag>path:name to project it).
  • a | b — fallback: show the first alternative that has a value (e.g. tag>label | tag>path:name). Optional parentheses group it into one column: (label | path:name).
  • &uuid / &version — metarecord metadata.

An empty columns box restores the defaults.

Working with results

Move the cursor with the arrow keys (or j/k); results load progressively as you scroll. Bulk field edits apply to the whole result set through the daemon's set/append/remove operations. Pair this panel with the detail panel to inspect one record.
